World 04 — Vibe Coding

Building with
AI as the engine.

Vibe coding is how I describe using AI tools to design, write and deploy entire websites without the traditional development cycle. The vision is mine. The execution happens at a pace that was not possible three years ago.

Claude AI Cursor Lovable Bolt GitHub Pages Framer
Scroll to explore

Design first.
Ship immediately.

Vibe coding is not about letting AI do everything. It is about removing the bottleneck between design intent and deployed product. When I have an idea for a product, I no longer have to wait weeks for a developer to build it. I describe what I want, iterate in real time, and the product goes live the same day.

The design thinking, the product decisions, the UX reasoning, all of that is still entirely human. What changes is the speed of translation from concept into working code. AI handles the syntax. I handle the soul.

Every project in this section was designed, built and deployed by me alone using AI coding tools. They are real, live products. Not mockups. Not prototypes.

How It Happens

The Vibe Coding Process

01

Define the Product

Every project starts with clarity about what it is, who it is for and what one thing it does better than anything else. No ambiguity before the build begins.

02

Prompt the Structure

I write a detailed prompt for the AI describing the layout, functionality and visual direction. The AI generates the first working version.

03

Iterate the Design

Using Cursor or continued conversation with Claude, I refine the visuals, fix interactions, adjust spacing and push the aesthetic until it matches the vision.

04

Deploy and Ship

The finished site goes live on GitHub Pages, Netlify or Vercel. The whole process from blank canvas to deployed URL typically happens in a single day.

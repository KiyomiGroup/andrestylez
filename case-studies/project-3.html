Game UI Designer

Project Overview

Maliyo Games is one of Africa's leading mobile gaming studios creating games that celebrate African culture, characters, and stories. Safari City is one of their flagship titles: a city-building mobile game set in an African urban environment.

My role as Game UI Designer was to redesign and enhance the game's UI system including the HUD (heads-up display), menus, icon system, and in-game interface components with a focus on clarity, visual quality, and African aesthetic identity.

"Game UI has one job: get out of the way of the game while still being beautiful."

Problem Statement

The existing Safari City UI was functional but had several areas that needed improvement to match the visual quality and polish expected by modern mobile game audiences:

  • Icon system lacked visual consistency different weights, styles, and levels of detail across the set
  • HUD elements were not optimised for different screen sizes and aspect ratios
  • Menu flows had unnecessary steps and unclear visual hierarchy
  • The African visual identity in the game world wasn't being reflected consistently in the UI layer
  • Some screens felt cluttered too many elements competing for visual attention

Design Process

Phase 1: Audit & Style Reference

I began by auditing all existing UI elements documenting inconsistencies in icon style, spacing, colour usage, and layout behaviour across screen sizes. I also studied reference games in the city-builder genre alongside African art and design references to establish a visual direction that felt both competitive and distinctly Maliyo.

Phase 2: Icon System Redesign

The icon system was the foundational design work. I established a consistent visual style: slightly rounded corners, a unified stroke weight, 2px optical corrections for small sizes, and a colour palette that pulled from the game's world palette (warm ochres, deep greens, earthy reds).

Icons were designed at 4 sizes: 16px, 24px, 32px, 48px with specific optical adjustments at each size rather than simply scaling.

Phase 3: HUD Redesign

The top-bar HUD was simplified removing low-priority elements and creating clear visual hierarchy between primary resources (coins, gems) and secondary indicators. Backgrounds were refined with subtle African-pattern-inspired textures that add personality without distracting from gameplay.

Phase 4: Menu Flows

The Store and Build Menu flows were reworked for clarity reducing the number of taps required for common actions and implementing a consistent tab system with clear active states.

Visual Design Decisions

  • Colour palette: Warm golds, earthy ochres, and deep greens grounded in African landscape colours while maintaining contrast for legibility
  • Typography: Rounded, bold display font for values and numbers; clean sans-serif for labels and descriptions
  • Texture: Subtle woven/pattern overlays on panels inspired by Kente and Adire textile patterns adding cultural texture without visual noise
  • Animation intent: Documented micro-animation specs for icon feedback (tap, unlock, collect) for developer implementation
  • Dark mode: All screens designed for dark UI more immersive, better battery on OLED, and consistent with genre conventions

Project Outcome

Full icon set redesigned across 5 categories, 4 sizes each

Unified design language applied across all UI elements
African visual language woven into UI patterns and colour

Design Reasoning & Reflection

Game UI design is a discipline that rewards restraint. Every pixel is competing with the actual game the world, the characters, the action. The best game UI is invisible until you need it, and satisfying when you interact with it.

Working on Safari City deepened my appreciation for icon design as a standalone craft it requires both illustration skills and systematic thinking. The constraint of designing at small sizes forces clarity of concept: if an icon doesn't read at 16px, the concept isn't strong enough.

The most rewarding part of this project was finding ways to embed African cultural references into the UI without making them feel forced or decorative the best moments were when the cultural identity emerged naturally from the design logic itself.

Project Overview

StreetTasker is a gig economy platform designed to connect everyday people with local workers who can get things done. Think of it as the missing middle layer between someone who needs help and someone nearby who has the time and skill to help them. The concept was born from a simple observation: Lagos is full of capable, hardworking people who just need a reliable way to find consistent work, and full of busy people who need small jobs done without the complexity of a formal hiring process.

This project was built entirely using vibe coding, which means I used AI tools throughout the entire process, from generating the initial structure, to writing the logic, to deploying the finished platform. The design decisions were still mine. The product thinking was still mine. But the speed of execution changed completely.

The Problem

Informal labor markets in Nigeria are massive but deeply fragmented. Workers rely on word of mouth, neighborhood networks and social media posts that disappear in hours. People who need help have no reliable way to find vetted, available workers nearby. Both sides of this equation are losing value every day because there is no platform built specifically for this context.

Existing platforms like Fiverr or TaskRabbit were not designed with the Nigerian market in mind. The pricing models, payment infrastructure and language patterns do not fit. StreetTasker needed to be something built specifically for the reality of how things work here.

User Flows

TASK POSTER FLOW
User lands on homepage
  → Sees clear value proposition
    → Clicks "Post a Task"
      → Fills in task name, description, location, budget
        → Task goes live on board
          → Workers browse and send interest
            → Poster picks a worker
              → Job gets done

WORKER FLOW
Worker lands on homepage
  → Clicks "Find Work"
    → Browses available tasks by category or location
      → Clicks a task to view details
        → Sends interest with a short message
          → Task poster responds
            → Work begins

Design Decisions

The interface uses a clean two-sided layout. The homepage immediately communicates both use cases side by side, so users can self-select into the right path without confusion. I used a dark blue palette to suggest trust and reliability, which matters a lot when people are making decisions about hiring someone they have never met.

Task cards are deliberately simple. Name of task, category, rough location, and a budget range. No clutter. The goal was to let workers scan quickly and act fast. Attention on these platforms is short and the design respects that.

Vibe Coding Process

This platform was built in a single extended session using Claude and Cursor. I started by describing the product concept and letting Claude generate the base HTML structure and component logic. Then I shaped the visual design by iterating on the CSS in Cursor, making changes in conversation with the AI rather than writing everything manually from scratch.

The result is a platform that would have taken a small team two to three weeks to build traditionally. Using vibe coding it was live in roughly two days. That speed does not compromise the quality. If anything, the tight feedback loop between me and the AI meant I could explore more design directions faster and land on better solutions.

Outcome

StreetTasker is live at streettasker.com. The platform demonstrates that a functional, well designed gig economy product can be scoped, designed and built by one person with the right tools. It stands as a proof of concept for what vibe coding makes possible, and as a real product that could scale with the right community behind it.

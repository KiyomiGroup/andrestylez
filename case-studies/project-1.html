Project Overview

Vitsetech is a technology-focused company with a functional desktop website but the mobile experience was inconsistent, cramped, and difficult to navigate on smaller screens. I was brought in to redesign and optimize the mobile responsiveness without rebuilding the site from scratch.

The goal was clear: make the existing website feel as polished and usable on a phone as it does on a desktop improving layout logic, touch usability, and readability at every breakpoint.

"A responsive site isn't just one that fits on a phone. It's one that feels designed for a phone."

Problem Statement

The Vitsetech website was built primarily with desktop users in mind. When accessed on mobile, several critical issues degraded the experience:

  • Layouts collapsed unpredictably at smaller breakpoints text overlapped, images broke out of containers
  • Text sizing and line-height was not adjusted for smaller screens, reducing readability
  • Interactive elements (buttons, links, forms) had inadequate touch target sizes below the recommended 44×44px minimum
  • Navigation was difficult to use on mobile the hamburger menu was inconsistent and slow to respond
  • Visual hierarchy broke down on narrow viewports sections that looked structured on desktop felt cluttered on mobile

Design Process

1. Audit & Analysis

I started by systematically reviewing the live site across five mobile breakpoints documenting every layout failure, spacing inconsistency, and touch usability issue. Each problem was logged with its location, severity, and recommended fix.

2. Mobile-First Restructuring

Rather than simply scaling down desktop layouts, I re-approached key sections with mobile-first thinking. This meant asking: "If this section was designed only for mobile, how would it be structured?" then building upward from there.

3. Component-Level UI Fixes

  • Adjusted all font sizes and line-heights for sub-768px viewports
  • Increased padding on buttons and tap targets to meet accessibility minimums
  • Converted side-by-side layouts to stacked single-column on mobile
  • Simplified navigation to a clean, accessible mobile menu
  • Optimized image display proper aspect ratios, no overflow

UX Audit Notes

These were the specific design decisions made during the audit and improvement phase:

  • Heading font sizes reduced from desktop (48px → 32px mobile) for better proportion
  • Section padding reduced from 80px to 40px on mobile preventing excessive whitespace
  • CTA buttons increased from 38px to 48px height ensuring accessible tap targets
  • Grid layouts (2-col, 3-col) converted to single column below 768px
  • Image containers constrained with max-width and border-radius consistency
  • Navigation z-index and backdrop fixed no more menu appearing behind content
  • Scroll performance improved by removing fixed-position elements that caused jank

Project Outcome

Significantly improved across all 5 tested breakpoints
All interactive elements meet minimum touch target standards
Consistent heading scale and spacing across all mobile views
Mobile design aligned with Vitsetech's existing visual identity

Design Reasoning Notes

This project reinforced an important principle: responsive design is not an afterthought. Mobile users are often in higher-intent moments they've found you through search or a referral link and they're making a quick judgment. If the site doesn't work on their device, that moment is lost.

The constraint of working within an existing site (rather than redesigning from scratch) required precision over creativity identifying exactly what was broken and fixing it systematically. This kind of audit-and-improve workflow is a core skill in professional UI/UX practice.
